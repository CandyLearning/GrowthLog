## 行為分類與句型骨架

### Creation

**When 句型模板**：
  `When 使用者{建立/新增/記錄}{實體}：`（接 Data Table）

**Then 句型模板（成功）**：
  `Then 操作成功`
  `Then {實體}應出現在列表中：`（接 Data Table）

**Then 句型模板（失敗）**：
  `Then 操作失敗，violation_type 為 "<violation>"`

### Mutation

**When 句型模板**：
  `When 使用者更新{操作}：`（接 Data Table）
  `When 使用者刪除感謝條目 $感謝條目.id`

**Then 句型模板（成功）**：
  `Then 操作成功`

**Then 句型模板（失敗）**：
  `Then 操作失敗，violation_type 為 "<violation>"`

### Query

**When 句型模板**：
  `When 使用者查詢{實體列表}`

**Then 句型模板**：
  `Then 查詢結果應包含以下{實體}：`（接 Data Table）
  `Then 查詢結果應為空`

## 共用契約

### Postcondition 模式

| 模式名稱 | 描述 | 適用操作 |
|---------|------|---------|
| 建立成功 → 出現在列表 | 新建實體立即可被查詢到 | 建立目標、新增紀錄、記錄心情、新增感謝條目 |
| 狀態持久化 | 操作結果寫入 DB 後查詢可驗證 | 所有 Mutation |

### Violation 模式

| Violation | 觸發條件 | 適用操作 |
|----------|---------|---------|
| `MISSING_REQUIRED_FIELD` | 必填欄位為空/未提供 | 建立目標（title）、新增紀錄（title）、新增感謝條目（content）、記錄心情（mood_type）、建立寵物（pet_name, species）、編輯感謝條目（content 空） |
| `UNAUTHORIZED` | 操作不屬於自己的資源 | 更新目標狀態、編輯/刪除感謝條目 |
| `NOT_FOUND` | 目標資源不存在 | 新增紀錄（goal not found）、更新目標狀態（goal not found）、編輯/刪除感謝條目（entry not found） |
| `INVALID_STATUS_TRANSITION` | 非法的 goal_status 轉換 | 更新學習目標狀態 |
| `ALREADY_EXISTS` | 重複建立唯一資源 | 建立電子寵物（一人一隻） |
| `INVALID_VALUE` | 欄位值不在合法範圍 | 記錄心情（非法 mood_type）、建立寵物（非法 species） |

## 共用句型

> Stage 2.5 萃取結果：以下 4 個句型跨全部 6 個 domain 共用。
> 各 domain 的句型.md 中已標記引用。

| # | 類別 | 句型 | Handler | 說明 |
|---|------|------|---------|------|
| G1 | Given | `Given 系統中有以下使用者：` | aggregate-given | 注入 User 資料至 DB |
| G2 | Given | `Given 以 $使用者.user_id 身份發出請求` | States Prepare | 設定 Authorization header（JWT） |
| T1 | Then | `Then 操作成功` | success-failure | HTTP 2xx 成功回應 |
| T2 | Then | `Then 操作失敗，violation_type 為 "<violation>"` | success-failure | HTTP 4xx，body 含 violation_type |
