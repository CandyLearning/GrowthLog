# auth 句型模型

## 實體概述

- **性質**: Aggregate
- **攜帶狀態**: google_id, display_name, avatar_url
- **Invariant 清單**: google_id 唯一（全域）；每個 google_id 對應唯一 User

## 操作清單

| 操作 | 行為分類 | When Data Table 欄位 | Then Data Table 欄位 | Violation Types |
|------|---------|---------------------|---------------------|-----------------|
| Google OAuth 登入 | Creation / Mutation | google_id, display_name, avatar_url | google_id, display_name, avatar_url | — |
| 更新個人資料 | Mutation | display_name?, avatar_url? | display_name, avatar_url | UNAUTHORIZED |

## 句型清單

> G1、G2、T1（操作成功）、T2（操作失敗）為共用句型，見系統抽象 G1-G2、T1-T2。

| # | 類別 | 句型 | Handler | 參數槽 |
|---|------|------|---------|-------|
| G1 | Given | `Given 系統中有以下使用者：` | aggregate-given | Data Table: user_id, google_id, display_name, avatar_url |
| G2 | Given | `Given 以 $使用者.user_id 身份發出請求` | States Prepare | $使用者.user_id |
| W1 | When | `When 使用者以以下 Google 帳號資訊完成 OAuth 授權：` | command | Data Table: google_id, display_name, avatar_url |
| W2 | When | `When 使用者更新個人資料：` | command | Data Table: display_name?, avatar_url? |
| T1 | Then | `Then 操作成功，回傳 session token` | success-failure | — |
| T2 | Then | `Then 操作成功` | success-failure | — |
| T3 | Then | `Then 操作失敗，violation_type 為 "<violation>"` | success-failure | \<violation\> |
| T4 | Then | `Then 系統中應存在以下使用者：` | aggregate-then | Data Table: google_id, display_name, avatar_url |
| T5 | Then | `Then 系統中使用者的個人資料應為：` | aggregate-then | Data Table: display_name, avatar_url |

## 測試分析

### Google OAuth 登入

**等價類劃分**: 首次登入（google_id 不存在 → 建立 User）/ 回訪登入（google_id 已存在 → 識別 User）  
**邊界值分析**: 不適用（OAuth 結果為離散狀態）  
**狀態轉移**: User 不存在 → 建立並回傳 token；User 存在 → 識別並回傳 token  
**組合覆蓋**: 不適用  
**錯誤推測**: OAuth 授權失敗由 Phase 05 實作定義，本階段不測試

**Examples 規劃**:

| # | 驗證目標 | 額外前置狀態 | 輸入資料摘要 | 預期結果 | 分析維度 |
|---|---------|-------------|-------------|---------|---------|
| 1 | 首次登入 → 建立 User + 回傳 token | （無） | google_id=gid-001（新） | 成功，token，User 建立 | 狀態轉移 |
| 2 | 回訪登入 → 識別 User + 回傳 token | User 已存在 | google_id=gid-001（已存在） | 成功，token | 狀態轉移 |

### 更新個人資料

**等價類劃分**: 更新 display_name / 更新 avatar_url  
**邊界值分析**: 不適用（長度限制由 Phase 04 API Contract 定義）  
**狀態轉移**: 不適用  
**組合覆蓋**: 不適用（兩欄位各自獨立選填，無交互副作用）  
**錯誤推測**: 未登入用戶請求 → UNAUTHORIZED

**Examples 規劃**:

| # | 驗證目標 | 額外前置狀態 | 輸入資料摘要 | 預期結果 | 分析維度 |
|---|---------|-------------|-------------|---------|---------|
| 1 | 更新顯示名稱成功 | 已登入 | display_name=新名字 | 成功，名稱更新 | 等價類 |
| 2 | 更新大頭照成功 | 已登入 | avatar_url=new/path.jpg | 成功，頭像更新 | 等價類 |
| 3 | 未登入更新失敗 | （無登入狀態） | display_name=xxx | UNAUTHORIZED | 錯誤推測 |
